## Page Composition & Layout

### Paper Size
- US Letter (8.5 x 11 inches) preferred; A4 acceptable for international distribution

### Margins
- Top: ~1.0 inch (space for header)
- Bottom: ~0.8 inch (space for footer)
- Left: ~1.0 inch
- Right: ~1.0 inch

### Column Layout
- **Cover page**: Two-column - main content (left ~60%), sidebar (right ~40%)
- **Body pages**: Single column, full width
- **Table of Contents**: Single column with right-aligned page numbers

### Whitespace & Rhythm
- Paragraph spacing: 4-6pt after each paragraph (dense institutional style)
- Section heading spacing: 10pt before, 4pt after
- Exhibit spacing: 10pt before and after
- **High information density** - institutional research prioritizes content density over whitespace
- Body text fills most of the page width; minimal margins for maximum content

## Typography System

### Reference Font Families (from PDF analysis)
- **Primary**: Univers LT Std (Regular, Light, Bold)
- **Labels**: Roboto Condensed (Regular, Bold)
- **Charts**: Arial (Regular, Bold)

### Substitute Font Strategy
When Univers is unavailable:
- **Latin-only**: Use Helvetica Neue, Arial, or Calibri as primary; maintain clean sans-serif character
- **CJK content**: Use Noto Sans CJK, Source Han Sans, or PingFang SC consistently for ALL text elements (headings, body, tables, captions, headers, footers, cover text, chart labels)

## Cover Page Design

### Critical Requirements (per judge feedback)
The cover page must be **dense and content-rich**, NOT minimalist. It contains:

### Top Banner
- Full-width navy blue (`#003B5C`) horizontal bar, height ~100-120pt
- Left side: Date in white text, 9-10pt (e.g., "April 25, 2017")
- Right side: Firm name/logo in white (e.g., "Goldman Sachs")
- Banner spans full page width (edge to edge)

### Title Block (inside banner, lower portion)
- Report title in large white bold text, ~30-32pt
- Title left-aligned within banner
- May span two lines if long

### Below Banner
- Category label right-aligned: report category (e.g., "Options Research")
- Subtitle in large dark charcoal text, ~18pt, positioned below banner area
- Subtitle is descriptive and sets context (e.g., "VIX ETPs: Popular, but often misunderstood")

### Two-Column Layout (below subtitle)
- **Left column (~60%)**: Executive summary with H2-style section headings in medium blue (`#4472C4`), followed by body text paragraphs
- 4-5 brief summary sections covering key topics (2-4 sentences each)
- **Right column (~40%)**: Sidebar with light gray (`#F2F2F2`) background containing:
  - Analyst names, credentials (Ph.D.), phone, email, firm
  - "N Things to Know" or "Key Takeaways" numbered list with bold lead-ins
  - Sidebar headings in medium blue (`#4472C4`)

### Bottom Disclaimer
- Full-width legal text in small font (8pt)
- Disclaimer in dark gray, spans multiple lines
- Above disclaimer: thin horizontal rule separating from main content
- Below disclaimer: firm name line (e.g., "The Goldman Sachs Group, Inc.")

### Left Sidebar Watermark (optional)
- Some institutional reports have a vertical text watermark on the left margin
- Small gray text, e.g., "For the exclusive use of [client]"

## Header & Footer

### Header (on all pages except cover)
- Separated from body by thin horizontal rule
- **Left**: Report date (e.g., "April 25, 2017")
- **Right**: Region/sector tag (e.g., "Global", "Americas", "Technology")
- Font: 8pt, gray (`#636363`)

### Footer (on all pages except cover)
- Separated from body by thin horizontal rule
- **Left**: Firm name + division (e.g., "Goldman Sachs Global Investment Research")
- **Right**: Page number
- Font: 8pt, gray (`#636363`)

## Tables

### Table Structure
- Every table is an "Exhibit" with sequential numbering: "Exhibit N: [Title]"
- Exhibit label appears above the table in bold
- Optional description paragraph below exhibit label, before table

### Header Row Style (IMPORTANT - per judge feedback)
- Background: **light blue-gray** (`#D6DCE4`) - a lighter, more subtle shade
- Text: **dark charcoal/black** (`#333333`), bold, 8-9pt (NOT white text)
- This creates a softer, more refined look than dark navy headers
- May use two-tier header with merged cells for grouped columns
- Thin top and bottom borders on header row

### Data Rows
- Alternating: white and light gray (`#F2F2F2`)
- Text: 8-9pt, regular weight
- Numbers: right-aligned; text: left-aligned
- Thin gray borders between rows (1px `#D9D9D9`)

### Footer Row
- Optional separator line above footnotes
- Footnotes: 7-8pt, gray, italic for sources

### Source Line
- Below table: "Source: [source name], [firm] Global Investment Research."
- Italic, 8pt, gray

## Charts & Exhibits

### Exhibit Numbering
- Sequential across entire document: Exhibit 1, Exhibit 2, ...
- Both charts and tables share the same numbering sequence
- Format: "Exhibit N: [Descriptive Title]" in bold above the exhibit

### Chart Container
- Encased in thin gray border
- Title bar at top: gray background (`#D9D9D9`), dark text, bold
- Chart area below title bar

### Chart Style
- **Bar charts**: Solid fill with navy/blue palette; no gradients; thin borders
- **Line charts**: 1.5-2pt line width; navy for primary series, gray for reference
- **Area charts**: Light fill with semi-transparent overlay
- **Pie charts**: Blue-gray palette, percentage labels
- **Axis labels**: 8-9pt, gray, outside axis
- **Gridlines**: Light gray horizontal only (`#E7E6E6`), no vertical gridlines
- **Legend**: Top-right or top-center, 8pt, gray text
- **Data labels**: On-chart values in 8-9pt when space permits

### Source Line
- Below chart container: "Source: [data source], [firm] Global Investment Research."
- Italic, 8pt, gray

## Bullet Points & Lists

### Bullet Style
- Filled circle bullets (solid black circles, ~4pt)
- Indent: ~0.3 inch from left margin
- Bullet-to-text spacing: ~0.15 inch

### Bold Lead-In Pattern
- Common pattern: **Bold phrase at start** followed by regular explanation
- Example: "**Long volatility: The long-run perspective.** The S&P 500 VIX Short-Term Futures ER Index..."
- Bold portion acts as a mini-heading within the bullet

### Numbered Lists
- Used for sequential items (e.g., "10 Things to Know")
- Arabic numerals with period: "1.", "2.", etc.
- Same bold lead-in pattern as bullets

## Mathematical Formulas

### Displayed Formulas (CRITICAL - per judge feedback)
- Left-aligned with descriptive label on the left
- Formula on the right, aligned with equals signs
- Clean spacing between terms
- Use a consistent tabular or aligned layout
- Example layout:
  ```
  VIX futures portfolio value yesterday:   P(t-1) = w(t-1) * F1,t-1 + (1-w(t-1)) * F2,t-1
  ```

### Sample Calculation Format
- Present step-by-step with labels on left and equations on right
- Show substitution of actual numbers
- Example:
  ```
  VIX futures portfolio as of the close         P(t-1) = w(t-1)*F1,t-1 + (1-w(t-1))*F2,t-1
  of trading January 23, 2017
                                                = 0.80 x 13.575 + 0.20 x 14.975 = 13.855
  ```

## Information Density

### Critical Requirement (per judge feedback)
- The reference document is **35 pages** of dense institutional research
- Each page should contain substantial text content
- Minimize unnecessary whitespace between paragraphs and sections
- Use the full page width effectively
- Exhibits should be sized appropriately (not oversized)
- Typical page has 400-500+ words of body text
- Section transitions should be tight with minimal spacing

### Content Density Guidelines
- Aim for **high word count per page** (institutional style)
- Keep paragraph spacing tight (4-6pt)
- Section headings should have minimal spacing above/below
- Exhibits should not dominate the page unless necessary
- Text should flow continuously without excessive breaks

## Font Handling Rules

### CJK / Mixed Script Content
- If content may contain CJK characters, use a single CJK-capable font for ALL text
- Do NOT use different font families for different components
- Recommended: Noto Sans CJK or Source Han Sans (weights: Regular, Bold)
- These fonts adequately preserve the clean, institutional sans-serif character of the reference

### Table Cell Handling
- CJK text in table cells must wrap properly
- Cells must grow vertically to accommodate wrapped text
- Line height: at least 1.2x font size for readability
- Never allow text to clip, overlap adjacent cells, or overflow columns

---

## What to AVOID (per judge feedback)

### Do NOT Use:
1. **Colored callout boxes** with left borders (e.g., "Key Finding", "Critical Distinction", "Warning") - these do not exist in the reference
2. **Minimalist cover design** - the cover must be dense and content-rich
3. **Horizontal rules as section separators** within body text - use whitespace sparingly
4. **Gradient backgrounds** on cover - use solid navy blue
5. **Large decorative elements** or excessive whitespace
6. **"Figure X:" labeling** - always use "Exhibit N:"
7. **Dark navy table headers with white text** - use lighter blue-gray headers with dark text
8. **Low information density** - pack content tightly
