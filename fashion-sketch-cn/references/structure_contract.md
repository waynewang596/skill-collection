## Table of Contents

- "Table of Contents" as H1 with coral underline
- Two-level hierarchy
- Main sections (Level 1) with **right-aligned page numbers** connected by **dotted leader lines**
- Subsections indented with dotted leaders to page numbers
- Style names listed individually under "Style Specifications"
- **Must fit on a single page** -- use compact spacing if needed

## Section Divider Pages

Before each of these major sections, include a clean divider page:
- **Style Specifications** (after Collection Overview)
- **Fabric & Trim Library** (after Style Specifications)
- **Bill of Materials (BOM)** (after Fabric & Trim Library)
- **Packaging & Labeling** (after BOM)
- **Quality Standards** (after Packaging & Labeling)

Format:
- H1 title with coral underline
- One brief description sentence below
- No other content

## Section 1: Collection Overview

### 1.1 Style Summary
- Introductory paragraph describing collection positioning
- Style cards in 2-column grid layout (5 rows for 10 styles)
- Each card contains:
  - Style code (coral colored) + Style name
  - Fabric: [material]
  - Category: [category]
  - Price Tier: [tier]

### 1.2 Size Chart Standards
- Introductory paragraph about sizing methodology
- **Table 1**: Standard Body Measurement Chart (inches)
  - Columns: Size, Chest, Waist, Hip, Shoulder, Sleeve
  - Rows: XS, S, M, L, XL
- **Measurement Tolerance Standards** callout box:
  - Critical: +/- 0.5 inch
  - Secondary: +/- 0.25 inch
  - Minor: +/- 0.125 inch

## Section 2: Style Specifications

Each style follows an identical **2-page template**:

### Page 1: Style Header + Flat Sketch + Primary Measurements

#### Style Header
- Style code + name as H2 (e.g., "JT-001 Wool Peacoat")
- Metadata bar with 3 columns:
  - CATEGORY / [category value]
  - PRIMARY/SHELL FABRIC / [fabric value]
  - LINING/FILL/ORIGIN / [value]

#### Flat Sketch Figure
- **Must be actual generated flat sketch illustrations** -- use the image generation tool
- Side-by-side front and back flat sketches, centered
- Caption: "Figure N: [Code] Front & Back Flat Sketch"
- Figures numbered sequentially across all styles (Figure 1 through Figure 11)

#### Construction Details Table
- Title: "Construction Details" as H3
- Columns: Component, Specification, Notes
- 5-8 rows per style covering: Front Closure, Collar, Pockets, Cuffs, Hem, Vent, Interior, etc.

#### Measurement Chart (Primary)
- Title: "Measurement Chart with Tolerances"
- Table label: "Table N [Code] Garment Measurements (inches)"
- Columns: Measurement Point, XS, S, M, L, XL, Tolerance
- Primary measurements (5 rows): Chest, Center Back Length, Sleeve Length, Shoulder Width, Sweep

### Page 2: Secondary Measurements + Colorways

#### Measurement Chart (Secondary)
- Continuation of measurement table from page 1
- Secondary measurements (2 rows): style-specific details (collar height, pocket opening, etc.)

#### Colorways Table
- Title: "Colorways" as H3
- Columns: Color Code, Color Name, Pantone Ref. (or Description)
- 3-4 colorway rows per style

**Page density rule**: Page 2 should contain both secondary measurements AND colorways. Do not split these across separate pages.

## Section 3: Fabric & Trim Library

### 3.1 Fabric Swatches
- Introductory sentence
- "Figure 11 Fabric Swatch Reference"
- **2x4 image grid of actual fabric swatch photographs** -- use the image generation tool to create fabric texture images
- Each swatch labeled: Code, Material Name, Available Colors
- **Fabric Specifications Table** below the swatch grid:
  - Columns: Code, Material, Weight, Width, Origin, Care

### 3.2 Hardware & Trims
Organized into sub-subsections with individual tables:

**Zippers Table**
- Columns: Style, Size, Type, Color Options, Application

**Buttons Table**
- Columns: Type, Size, Material, Color, Application

**Knit Trims Table**
- Columns: Type, Gauge, Width, Fiber Content, Application

**Labels & Tags Table**
- Columns: Item, Material, Size, Printing, Placement

## Section 4: Bill of Materials (BOM)

- Introductory sentence
- One BOM per representative style (not all styles need full BOM)

### BOM Table Structure
- Header row: **dark navy (#2D3142) background with white text** (not gray)
- Columns: Item, Description, Color, Qty, Unit, Supplier
- Category divider rows: FABRICS, HARDWARE, LABELS & TRIMS
- **BOM Notes** callout box:
  - Yield Allowance note
  - Color Matching tolerance
  - Supplier Approval requirement

## Section 5: Packaging & Labeling

### 5.1 Individual Garment Packaging Table
- Columns: Item, Specification, Material, Notes

### 5.2 Carton Specifications Table
- Columns: Carton Type, Size, Max Weight, Qty/Style, Material

### 5.3 Carton Labeling
- **Required Carton Markings** callout box:
  - Shipping Label requirements
  - Barcodes
  - Handling instructions
  - Origin marking

## Section 6: Quality Standards

### 6.1 AQL Standards Table
- Columns: Inspection Level, Critical, Major, Minor, Sample Size
- Rows: Normal, Tightened, Reduced

### 6.2 Critical Defects
- Intro sentence: "Zero tolerance items"
- Bullet list: needle/sharp objects, flammable materials, restricted substances, incorrect care labeling, loose buttons

### 6.3 Major Defects
- Bullet list: seam slippage, incorrect sizing, color variation, visible damage, misaligned patterns, non-functional closures

### 6.4 Minor Defects
- Bullet list: loose threads, press marks, puckering, label placement variance

### 6.5 Testing Requirements Table
- Columns: Test, Standard, Requirement, Frequency
- Rows: Colorfastness to Light, Colorfastness to Crocking, Seam Strength, Tear Strength, Dimensional Stability, Flammability
- **Special Notes for Leather & Fur** callout box

### 6.6 Final Inspection Checklist Table
- Columns: Checkpoint, Standard, Pass/Fail
- 9 checkpoints: appearance, stitch quality, seam allowances, button/snaps, zipper function, measurements, cleanliness, labels, packaging

## Approval & Sign-Off (Part of Section 6)

### Introductory paragraph
Statement that all parties must review and approve before production.

### Signature Blocks
- 3x2 grid layout
- Roles: Design Director, Technical Director, Production Manager, Factory Representative, Quality Manager, Merchandising
- Each: title + signature line + date line

### Revision History Table
- Columns: Version, Date, Description, Author

### Confidentiality Notice
- "CONFIDENTIAL" label
- Proprietary information statement
- Document metadata line

## Cross-Reference and Numbering Systems

### Figure Numbering
- Sequential across entire document: Figure 1, Figure 2, ... Figure 11
- Format: "Figure N: [Style Code] [Description]"

### Table Numbering
- Sequential across entire document: Table 1, Table 2, ... Table 11+
- Format: "Table N [Style Code] [Description]"

### Style Code Format
- Pattern: JT-NNN (e.g., JT-001, JT-002)
- Color Code Format: NNN-XXX (e.g., 001-NVY, 002-BLK)
- Fabric Code Format: F-NNN (e.g., F-001, F-002)

## Page Break Rules

- Cover page: standalone
- Table of Contents: standalone (single page)
- Each section divider page: standalone
- Each H1 section starts on a new page
- Each individual style spec starts on a new page (2 pages per style)
- BOM for each style starts on a new page
- Approval & Sign-Off: starts on new page after Quality Standards

## Page Count Target

For a collection of 10 styles, target approximately 33 pages:
- Cover: 1 page
- TOC: 1 page
- Collection Overview: 2 pages (1 page style cards + 1 page size chart)
- Section Dividers: 5 pages
- Style Specifications: 20 pages (10 styles x 2 pages)
- Fabric & Trim Library: 2 pages (1 page swatches + specs + 1 page hardware)
- BOM: 1-2 pages
- Packaging & Labeling: 1 page
- Quality Standards + Approval: 2 pages

## Adaptation Rules

1. **Style count**: The reference has 10 styles. Adapt the template for any number of styles (5-20 typical).
2. **Product category**: Reference is jackets. Adapt for any apparel category (pants, shirts, dresses, accessories) by adjusting the metadata bar labels and measurement points.
3. **BOM scope**: Reference shows partial BOMs (representative styles). For full collections, include BOMs for all styles or a summary BOM table.
4. **Fabric swatches**: Generate AI fabric texture images for the swatch grid. If unavailable, use the Fabric Specifications table alone.
5. **Flat sketches**: Generate AI flat sketches for each style. If sketches cannot be generated, describe the style thoroughly in the construction details.
